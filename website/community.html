Ways to Contribute

Report Bugs

  • Search existing issues first
  • Provide clear description
  • Include steps to reproduce
  • Add error messages and logs
  • Run axle doctor and include output

Suggest Features

  • Check if feature already exists
  • Describe the use case clearly
  • Explain why it's important
  • Suggest possible implementation
  • Consider if it fits the project scope

Add Tools

  • Follow tool contract (get_description, main)
  • Include error handling
  • Add docstrings
  • Test thoroughly
  • Submit pull request

Improve Docs

  • Fix typos and grammar
  • Add examples
  • Clarify confusing sections
  • Update for new features
  • Translate to other languages

Test Features

  • Test on your platform
  • Report edge cases
  • Verify fixes work
  • Check documentation accuracy
  • Provide feedback on UX

Help Others

  • Answer questions on GitHub
  • Share your tools
  • Write tutorials
  • Mentor new contributors
  • Spread the word

Community Guidelines

We want to ensure a welcoming and productive community for everyone. Please follow these guidelines:

Be Respectful

  • Treat everyone with respect and professionalism
  • Welcome newcomers and help them learn
  • Assume good intentions from others
  • Be constructive in your feedback

Be Collaborative

  • Work together to solve problems
  • Share knowledge and experience
  • Ask for help when you need it
  • Offer help when you can provide it

Be Inclusive

  • Respect different viewpoints and experiences
  • Use inclusive language
  • Accept that not everyone will agree with you
  • Focus on what is best for the community

Getting Started

  • Check existing documentation before asking
  • Search GitHub issues before creating new ones
  • Start with good practices: clear titles, detailed descriptions
  • Be patient - maintainers have limited time
